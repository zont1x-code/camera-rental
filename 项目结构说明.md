# Camera Rental 项目结构说明

## 项目概述

基于 **Node.js + Express** 的相机租赁预约网站，前端使用原生 HTML/CSS/JS，数据以 JSON 文件存储在 `data/` 目录。

启动方式：`npm start` → 访问 `http://localhost:3000`
管理后台：`http://localhost:3000/admin/login.html`（账号 `admin` / `admin123`）

---

## 全局前置功能

### 隐私协议弹窗

用户首次访问任意页面时，检测 cookie `privacy_agreed`。若无，弹出全屏遮罩展示隐私协议内容（通过 `GET /api/privacy` 实时获取）。两个按钮：

- **同意并继续** — 设 cookie 有效期 30 天
- **不同意** — 无法使用平台

### 手机号轻量登录

隐私同意后，导航栏右侧显示手机号输入框。输入 11 位手机号后调用 `POST /api/login`，自动创建或识别用户，登录态存入 `localStorage`。

### 导航栏

白色背景固定顶部，三个入口：首页（预约）| 发现 | 我的预约。当前页高亮橙色。

## 各页面说明

### [public/index.html](public/index.html) — 预约首页

三步预约流程：

1. **选择机型** — 三台相机卡片（图片 / 型号 / 优势文案），选中显示橙色边框
2. **选择时间** — 展开日历组件，已预约日期灰色禁用，选区间橙色高亮，最多 3 天
3. **填写信息** — 姓名、学校、备注（选填）

右侧固定价格面板实时显示费用。提交时自动携带手机号，成功后跳转 `success.html`。

### [public/discover.html](public/discover.html) — 发现页

三个板块：

- **官方精选区** — 4 张轮播实拍占位图 + 三台相机详细参数 + 租赁流程步骤条（选机 → 预约 → 取机 → 归还）
- **社区分享区** — 展示已审核通过的帖子（图片 / 描述 / 发布者手机号脱敏如 138****），上方 **我要分享** 按钮弹出表单提交
- **公告栏** — 拉取 `GET /api/announcements` 展示

### [public/my-bookings.html](public/my-bookings.html) — 我的预约

根据当前登录手机号查询所有订单。每条显示：

- 相机型号、日期区间、姓名、学校
- 状态标签：待确认（灰）/ 已确认（蓝）/ 租赁中（橙）/ 已完成（绿）/ 续约待确认（黄）
- 对「已确认」和「租赁中」订单，显示 **续约** 按钮 → 弹出日历选新日期（≤3 天），提交后生成续约申请等待管理员审核

### [public/success.html](public/success.html) — 预约成功

简约成功提示页，含返回首页和查看我的预约两个入口。

## 管理后台（admin/dashboard.html）

顶部 4 个 Tab 切换：

### 订单管理

- 表格展示所有预约，支持按状态筛选（全部 / 待确认 / 已确认 / 租赁中 / 已完成 / 续约待确认）
- 可手动修改订单状态（下拉选择）
- 续约申请行高亮，带 **通过** / **拒绝** 按钮
- 支持 CSV 导出

### 内容审核

- 帖子列表，可按状态筛选（全部 / 待审核 / 已通过 / 已拒绝）
- 每条显示图片缩略图、描述、发布者（脱敏）、时间
- 待审核帖子可 **通过** 或 **拒绝**

### 公告管理

- 输入框 + 添加按钮发布新公告
- 现有公告列表，可逐条删除

### 隐私协议

- 多行文本框加载当前内容
- 保存后实时更新用户端弹窗内容

## 后端接口清单

### 用户端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/cameras` | 获取相机列表 |
| GET | `/api/bookings?cameraId=` | 获取某相机的已预约日期区间 |
| POST | `/api/bookings` | 创建预约 `{ cameraId, startDate, endDate, name, school, remark, phone }` |
| POST | `/api/login` | 手机号登录 `{ phone }`，自动创建用户 |
| GET | `/api/my-bookings?phone=` | 查询该手机号的所有订单（含相机型号） |
| POST | `/api/renew` | 续约申请 `{ orderId, newStartDate, newEndDate }` |
| GET | `/api/privacy` | 获取隐私协议内容 |
| GET | `/api/announcements` | 获取公告列表 |
| GET | `/api/posts?status=approved` | 获取已审核通过的帖子（手机号脱敏） |
| POST | `/api/posts` | 提交帖子 `{ phone, imageUrl, description }`，默认 pending |

### 管理端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/admin/login` | 管理员登录 `{ username, password }` |
| GET | `/admin/api/bookings?status=` | 获取所有订单（含相机型号），可按状态筛选 |
| PUT | `/admin/api/bookings/:id` | 更新订单状态 `{ status }` |
| DELETE | `/admin/api/bookings/:id` | 删除订单 |
| PUT | `/admin/api/renew/:id/approve` | 通过续约申请（更新订单结束日期） |
| PUT | `/admin/api/renew/:id/reject` | 拒绝续约申请 |
| GET | `/admin/api/posts?status=` | 获取帖子列表，可按状态筛选 |
| PUT | `/admin/api/posts/:id/approve` | 审核通过帖子 |
| PUT | `/admin/api/posts/:id/reject` | 拒绝帖子 |
| GET | `/admin/api/announcements` | 获取公告列表 |
| POST | `/admin/api/announcements` | 添加公告 `{ content }` |
| DELETE | `/admin/api/announcements/:id` | 删除公告 |
| GET | `/admin/api/privacy` | 获取隐私协议 |
| PUT | `/admin/api/privacy` | 更新隐私协议 `{ content }` |

## 订单状态流转

```
pending（待确认）→ confirmed（已确认）→ renting（租赁中）→ completed（已完成）
                       ↓                       ↓
                  renew_pending（续约待审核）→ 管理员通过 → 更新结束日期
                       ↓
                  管理员拒绝 → 恢复原状态
```
